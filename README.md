# UrbanTechStroy

Сайт на [Nuxt 3](https://nuxt.com/) + Tailwind. Деплой с **Node.js** (preset `node-server`) — нужен запущенный сервер, чтобы работали API и база. **Пошаговый деплой на VPS (Git, systemd, Nginx, обновления): см. [DEPLOY.md](./DEPLOY.md).**

### Превью в соцсетях (Open Graph)

Задайте **`NUXT_PUBLIC_SITE_URL`** (например `https://ваш-домен.ru`) — для корректного **абсолютного** адреса картинки `og:image` (используется **`/favicon.png`**, тот же файл, что и фавикон). При пустом значении на продакшене обычно подставляется origin из запроса. Текст превью: **`NUXT_PUBLIC_OG_DESCRIPTION`** или встроенное описание компании в `app.vue`.

## Админ-панель

URL: **`/admin`** (вход: **`/admin/login`**, регистрация первого пользователя: **`/admin/register`** — только если в БД ещё нет администраторов и задан **`NUXT_ADMIN_SETUP_KEY`**).

### Настройка на сервере

1. Скопируйте `.env.example` в `.env` или задайте переменные в панели хостинга. На **VPS с PM2** переменные из файла `.env` сами в процесс не попадают — используйте **`ecosystem.config.cjs`** из репозитория (см. **[DEPLOY.md](./DEPLOY.md)** → раздел «PM2»).

**Первый администратор (один из вариантов):**

- **Вариант A:** **`NUXT_ADMIN_PASSWORD`** + **`NUXT_ADMIN_EMAIL`** (обязательно email). При первом запросе к серверу в SQLite создаётся учётная запись. После изменения этих переменных **перезапустите Node/Nuxt** — пароль (и при необходимости email) подтянутся в БД: совпадение по email, либо если в базе **ровно один** администратор, его логин и пароль приводятся к значениям из `.env` (удобно, если сначала создался дефолтный `admin@urbantechstroy.local`). Пароль в БД хранится только в виде хеша.
- **Вариант B:** **`NUXT_ADMIN_SETUP_KEY`** — секретная строка. Откройте **`/admin/register`**, введите ключ, email и пароль — будет создан первый администратор и выполнен вход.

Дальнейшие администраторы добавляются в панели **«Администраторы»** (`/admin/admins`).

Прочее:

- **`NUXT_DATABASE_PATH`** (опционально) — путь к файлу SQLite. По умолчанию **`.data/site.db`**.
- **`NUXT_PUBLIC_SITE_URL`** (рекомендуется) — канонический адрес сайта без `/` в конце; для превью ссылок в соцсетях и `og:image`.
- **`NUXT_PUBLIC_OG_DESCRIPTION`** (опционально) — текст для Open Graph / Twitter.

2. Перезапустите приложение после изменения переменных.

3. Папку **`.data`** держите на постоянном диске (не сбрасывается при деплое), иначе лиды, правки контента, **учётные записи админов** и **загруженные картинки** пропадут. Добавьте volume / бэкап при необходимости.

### Возможности

- **Администраторы** — email (логин) и пароль: добавление, смена данных, удаление (нельзя удалить последнего и свою текущую учётную запись).
- **Услуги** — правки текстов и изображений (превью, замена файла, сброс, свой URL), **галерея** с миниатюрами и порядком, **скрытие** с сайта, **сброс** к коду, **удаление карточек из кода** (с возможностью вернуть в админке), добавление и удаление карточек только в БД.
- **Проекты** — то же: скрыть / сбросить / **удалить карточку из кода** / добавить / удалить вручную добавленные; изображения и галерея — как в услугах.
- **Лиды** — заявки из формы **«Оставить заявку»** в блоке «Свяжитесь с нами» и из формы **«Рассчитайте стоимость проекта»**.

Публичные API (без авторизации):

- `GET /api/content/services` — услуги с учётом переопределений.
- `GET /api/content/projects` — проекты с учётом переопределений.
- `POST /api/leads` — приём заявки (поля: `source`: `contact` | `calculator`, `name`, `phone`, опционально `message`, для калькулятора `metadata`).
- `GET /api/uploads/<uuid>.<ext>` — отдача загруженного изображения (используется в URL после загрузки из админки). После **«Сохранить всё»** в админке (услуги или проекты) файлы в каталоге загрузок, на которые больше нет ссылок в сохранённом контенте, **удаляются автоматически** (снимки с `/images/…` из репозитория не трогаются).

Админ (после входа, cookie):

- `POST /api/admin/upload` — загрузка одного файла (`multipart/form-data`, поле `file`), ответ `{ url }` (до 5 МБ, JPEG/PNG/WebP/GIF).

> Статический хостинг без Node (например, только GitHub Pages) **не подойдёт** для админки и БД — нужен процесс Node с Nitro.

## Локальная разработка

```bash
npm install
npm run dev
```

Задайте в `.env` **`NUXT_ADMIN_PASSWORD`** + **`NUXT_ADMIN_EMAIL`** или **`NUXT_ADMIN_SETUP_KEY`** для проверки входа в `/admin` (см. раздел «Админ-панель» выше).
